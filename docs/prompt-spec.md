# LLM提示词规范：剧本到分镜（Seedance 2.0 适配版）

## 1. 目的

这份规范用于把“原始剧本”稳定转成：

1. 剧本解析结果
2. 分集结构
3. 单集片段结构
4. 单集短镜头分镜
5. 可直接适配 `Seedance 2.0` 的镜头提示词

目标不是让 LLM 自由发挥，而是：

- 忠于原稿
- 分层输出
- 可审可改
- 后续可自动化
- 镜头提示词适配 `Seedance 2.0`
- 尽量降低人物漂移、场景漂移、说话人错误、镜头语言单一等问题

## 2. 总体原则

### 2.1 不要一条 prompt 一步到位

不要直接使用这种写法：

```text
你是专业短剧导演，把这个剧本拆成80集并拆成镜头。
```

这样常见问题是：

- 过度改编
- 台词乱改
- 节奏虚高但不忠于原稿
- 镜头模板化
- 分集和分镜混在一起

必须拆成多个阶段。

### 2.2 固定阶段

推荐固定为 6 个阶段：

1. 剧本解析
2. 分集规划
3. 资产规格规划
4. 单集片段拆分
5. 单集镜头拆分
6. `Seedance 2.0` 镜头提示词生成

### 2.3 统一硬约束

后续所有阶段都应带上这些约束：

- 不新增主要人物
- 不新增主线剧情
- 不改人物关系
- 可以调整切分方式，但不能改变事件因果
- 输出结构化结果
- 明确哪些是原稿事实，哪些是总结归纳

## 3. 推荐调用顺序

### 3.1 顺序

1. `Prompt A`：剧本解析
2. `Prompt B`：分集规划
3. `Prompt C`：资产规格规划
4. `Prompt D`：单集片段拆分
5. `Prompt E`：单集镜头拆分
6. `Prompt F`：Seedance 2.0 镜头提示词生成

### 3.2 不要跳步

尤其不要跳过：

- 资产规格规划
- 片段拆分
- 连续性约束
- 说话人绑定
- 角色/场景资产绑定

## 4. Prompt A：剧本解析

### 4.1 用途

目标是先理解剧本，不直接拆镜头。

### 4.2 建议提示词

```text
你是专业的古装短剧编剧统筹与分集策划，不是自由改编作者。现在请你研读我提供的原始剧本，严格忠于原稿，不新增人物、不新增剧情主线、不随意改写人物关系。

你的任务不是直接写分镜，而是先完成“剧本解析”：

1. 提取主要人物、身份、关系、外貌气质特征。
2. 提取故事主线、副线、情感线、权力线。
3. 提取关键剧情节点、反转点、爆点、悬念点。
4. 提取适合做分集切点的剧情断点。
5. 输出全剧时间线和角色状态变化线索。

要求：
- 忠于原稿，不做过度改编。
- 用结构化格式输出。
- 明确哪些内容是原稿已有，哪些只是你的归纳。
- 暂时不要拆镜头，不要补充台词，不要扩写剧情。

请按以下格式输出：

一、主要人物
二、人物关系
三、剧情主线与副线
四、关键剧情节点
五、适合分集的切点
六、角色状态变化线
七、原稿风险点（如剧情表达不清、场次切换突兀、角色状态容易断裂）
```

### 4.3 预期输出

- 人物表
- 人物关系表
- 全剧关键节点
- 分集切点候选
- 连续性风险点

## 5. Prompt B：分集规划

### 5.1 用途

把总剧本拆成适合短剧节奏的分集结构，但不直接拆镜头。

### 5.2 建议提示词

```text
你是专业竖屏短剧导演兼分集编剧。请基于已经整理好的剧本解析结果，把该剧本拆分成适合竖屏短剧的分集结构。

目标要求：
1. 全剧总集数不少于80集。
2. 每集成片时长不低于2分钟。
3. 每集必须有明确的开场钩子、本集推进、结尾悬念。
4. 分集必须忠于原稿，不新增主线剧情，不随意改写台词含义。
5. 允许为了短剧节奏调整场次切分，但不能改变人物关系和事件因果。

请输出每一集：
- 集数
- 对应原稿范围
- 本集开场钩子
- 本集核心冲突
- 本集情绪推进
- 本集结尾悬念
- 预计时长
- 关键出场人物
- 连续性提醒

要求：
- 不写镜头。
- 不扩写台词。
- 不额外发明剧情。
- 节奏要适合短剧，但保持原稿逻辑。
```

### 5.3 预期输出

- 每集标题或编号
- 每集钩子/冲突/悬念
- 每集时长预估
- 关键人物和连续性提示

## 6. Prompt C：资产规格规划

### 6.1 用途

在分集完成后、正式拆片段和拆镜头前，先定义“需要哪些资产”，但此时不立即生成。

### 6.2 目标

这一步不是出图，而是输出资产规格书和需求表。

要求 LLM 帮我们先列清楚：

- 需要哪些角色核心资产
- 需要哪些角色造型资产
- 需要哪些场景资产
- 需要哪些模板群演资产
- 哪些关键片段需要关键帧资产

### 6.3 建议提示词

```text
你是专业短剧导演兼资产统筹。请基于已经完成的剧本解析和分集规划，输出该剧在当前阶段所需的资产规格与需求表。

注意：
1. 这一步只列资产需求，不直接生成图片。
2. 目标是先约定资产体系，保证后续镜头执行时有统一的资产标准。
3. 请区分“全剧长期资产”和“当前若干集需要优先准备的资产”。

请至少输出以下几类资产需求：

一、角色核心资产
- 正面脸
- 侧面脸
- 半身标准像
- 全身标准像
- 骨相/肤色/气质/年龄感标签

二、角色造型资产
- 常服
- 特定剧情状态服装
- 发型
- 妆造
- 配饰
- 常用道具

三、场景资产
- 全景建立图
- 主活动区视角
- 门口/入口视角
- 对峙视角
- 过渡视角
- 时间状态变体

四、模板群演资产
- 侍女模板组
- 侍卫模板组
- 近臣模板组
- 刺客模板组
- 面首模板组

五、关键帧资产需求
- 片段首镜
- 三人以上关系镜
- 身份容易混淆的镜头
- 动作复杂镜头
- 悬念落点镜

请按以下格式输出：

1. 全剧资产规格总表
2. 当前优先集数的资产需求表
3. 每类资产的用途说明
4. 哪些资产必须在镜头执行前先准备
5. 哪些资产可以延后，按缺失再补

要求：
- 不直接生成图
- 不写提示词
- 只输出资产规格和需求结构
```

### 6.4 预期输出

- 全剧资产规格总表
- 当前优先集的资产需求表
- 关键帧需求表

## 7. Prompt D：单集片段拆分

### 7.1 用途

把单集拆成多个剧情片段，解决“场景转换生硬”和“镜头一上来就拆太碎”的问题。

### 7.2 建议提示词

```text
你是专业短剧导演。请把第X集拆成若干剧情片段，而不是直接拆成长镜头。

拆分目标：
1. 每个片段只承担一个明确剧情任务。
2. 每个片段内部后续可以拆成多个短镜头。
3. 片段之间要有明确衔接，不要硬切。
4. 保证整集节奏适合竖屏短剧，不要全是人物对话近景。
5. 保留原稿内容，不新增关键剧情。

每个片段必须输出：
- 片段编号
- 片段名称
- 片段功能（建立 / 试探 / 冲突 / 反应 / 过渡 / 悬念）
- 片段对应原稿内容
- 片段发生场景
- 片段涉及人物
- 片段的开头信息
- 片段的结尾落点
- 与上一片段的衔接方式
- 与下一片段的衔接方式

额外要求：
- 如果场景切换明显，必须加过渡片段或过渡镜需求说明。
- 如果人物关系复杂，优先拆出关系镜片段，不要挤在一个片段里。
```

### 7.3 预期输出

- Beat sheet
- 每个片段的功能和衔接方式

## 8. Prompt E：单集镜头拆分

### 8.1 用途

把片段拆成短镜头组，而不是长镜头。

### 8.2 时长建议

- 建立镜：`4-6秒`
- 动作镜：`3-6秒`
- 关系镜：`4-7秒`
- 反应镜：`3-5秒`
- 悬念镜：`2-4秒`

一般不要超过 `8 秒`，除非确实是建立空间或完整关系镜。

### 8.3 建议提示词

```text
你是专业竖屏短剧导演。请基于第X集已经完成的片段结构，把每个片段拆成多个短镜头。

要求：
1. 每个镜头尽量控制在3-8秒。
2. 每个镜头只承担一个明确任务，不要一个镜头同时完成太多事情。
3. 不要把整集都拆成近景特写。
4. 每个片段至少要考虑建立镜、关系镜、动作镜、反应镜、落点镜的搭配。
5. 保持空间、站位、人物调度和剧情逻辑连续。
6. 忠于原稿，不乱加戏。

每个镜头必须输出：
- 镜头编号
- 所属片段
- 镜头功能
- 时长
- 景别
- 机位
- 运镜
- 人物站位
- 动作描述
- 台词内容
- 说话人
- 声音提示
- 与上一镜的衔接
- 与下一镜的衔接
- 连续性约束

额外要求：
- 必须明确谁在说话，不能只写台词内容。
- 如果是多角色镜头，必须写清谁站在哪边，谁看向谁。
- 不允许全是正面近景对话镜。
```

### 8.4 预期输出

- 单集 shot list
- 每镜功能和衔接
- 明确说话人
- 明确站位

## 9. Prompt F：Seedance 2.0 镜头提示词生成

### 9.1 用途

把“镜头脚本”转换成适配 `Seedance 2.0` 的实际生成提示词。

### 9.2 这一步必须额外强调的事情

这一步不是普通导演文案，而是“能给视频模型稳定执行”的提示词。

所以必须强调：

- 适配 `Seedance 2.0`
- 明确人物身份
- 明确说话人
- 明确站位
- 明确场景
- 明确镜头时序
- 明确禁止项
- 每一个镜头必须绑定角色资产和场景资产

### 9.3 建议提示词

```text
你是专业竖屏短剧导演，同时非常熟悉 Seedance 2.0 视频生成模型的提示词写法。请把我提供的单镜头脚本，转换成适配 Seedance 2.0 的高可靠视频生成提示词。

目标：
1. 提示词必须适合 Seedance 2.0 生成稳定视频。
2. 提示词必须降低人物漂移、场景漂移、说话人错误、口型错位、站位跳变、镜头语言单一的问题。
3. 每一个镜头都必须绑定角色资产和场景资产，确保人物和场景不会漂移。

请严格遵守以下规则：

一、镜头结构
- 先写全局画面风格和时代氛围。
- 再写场景参考和人物参考。
- 再写镜头功能。
- 再写固定站位。
- 再写景别、机位、运镜。
- 再按时间阶段写动作变化，例如 0-2秒、2-5秒、5-8秒。
- 再写台词和说话人。
- 最后写声音风格与禁止项。

二、Seedance 2.0 适配要求
- 提示词必须适合短视频镜头生成，不要写过于文学化的形容。
- 不能把太多动作、台词、人物关系塞进一个镜头。
- 每个镜头都要明确主要视觉中心。
- 每个镜头都要明确说话人，不能只写对白内容。
- 多角色镜头必须写清人物身份、服装、站位和视线方向。
- 每个镜头都必须重复绑定角色资产和场景资产。
- 若镜头需要连续性，可额外使用上一镜尾帧作为辅助，但不能代替角色和场景资产绑定。

三、人物和资产绑定要求
- 输出时必须写清本镜头需要绑定哪些角色资产。
- 输出时必须写清本镜头需要绑定哪个场景资产。
- 如果镜头中有多角色，必须分别点名身份，例如：
  - 李云湘：女性长公主，深绛红服
  - 谢风凌：男性黑衣侍卫
  - 顾文池：男性白衣执扇旧宠
- 必须写明排斥项，例如：
  - 顾文池不得生成女性
  - 白衣人物不是侍女，不是公主
  - 黑衣人物不是皇帝，不是顾文池

四、输出格式
请按以下结构输出：

1. 镜头编号
2. 角色资产绑定
3. 场景资产绑定
4. 是否使用上一镜尾帧辅助
5. Seedance 2.0 正式提示词
6. 台词
7. 说话人
8. 声音要求
9. 反向约束

现在请把以下镜头脚本转换成 Seedance 2.0 可用的正式生成提示词。
```

## 9. Seedance 2.0 镜头提示词专项约束

### 9.1 必须强调的硬规则

后续生成镜头提示词时，必须重复强调这些点：

- 每一个镜头都要绑定角色资产
- 每一个镜头都要绑定场景资产
- 多角色镜头必须写清身份、服装、站位、视线
- 必须明确说话人是谁
- 必须写明谁不能开口
- 必须有景别和机位
- 必须按时间阶段写动作
- 必须写禁止项

### 9.2 典型禁止项

可直接复用：

```text
不要现代元素，不要塑料质感，不要网红古偶感，不要统一古装男主脸，不要人物串位，不要身份互换，不要性别漂移，不要说话人错误，不要画面内字幕，不要logo，不要水印，不要漂浮文字，不要镜头乱晃，不要近景堆叠，不要过度慢动作，不要人物突然贴近，不要把白衣人物生成为女性，不要把黑衣侍卫生成为旧宠。
```

### 9.3 说话人绑定写法

例如：

```text
这句台词只能由谢风凌发声。
顾文池只能做无声反应，不得张嘴，不得代说。
嘴型只能出现在谢风凌脸上。
```

### 9.4 多角色关系镜写法

例如：

```text
固定站位：李云湘位于门口前景居中，谢风凌位于庭院右侧中景，顾文池位于庭院左侧中景。三人保持当前三角关系，不允许身份互换，不允许服装互换，不允许性别漂移。
```

### 9.5 观察镜写法

例如：

```text
顾文池位于前景右侧，身体和头部朝左前方练剑区，目光明确落在远处练剑的谢风凌身上。不得看镜头，不得看向右侧画外。
```

## 10. 推荐在项目中固化的 Prompt ID

建议后续在代码里固定成这几个 prompt：

- `script_analysis_v1`
- `episode_planning_v1`
- `beat_planning_v1`
- `shot_planning_v1`
- `seedance_shot_prompt_v1`

后续迭代时按版本升级，例如：

- `seedance_shot_prompt_v2`
- `seedance_shot_prompt_v3`

不要直接覆盖旧版，方便回溯问题。

## 11. 当前最重要的项目固化结论

### 11.1 分层 prompt 是必须的

不能把剧本解析、分集、片段、分镜、视频提示词一次做完。

### 11.2 镜头提示词必须适配 Seedance 2.0

不能只写导演语言，还要写模型可执行语言。

### 11.3 每个镜头必须绑定资产

必须写成系统级硬规则：

- 每一个镜头都绑定角色资产
- 每一个镜头都绑定场景资产
- 尾帧只能辅助，不能替代资产绑定

### 11.4 说话人必须结构化

后续项目里建议把每镜的说话人做成明确字段，而不是藏在提示词里。

例如：

- `speaker_ids`
- `silent_character_ids`
- `dialogue_lines`

### 11.5 最终建议

实际工程落地时，建议：

1. 先把以上 6 个 prompt 直接做成模板文件。
2. 每个 prompt 使用固定输入结构。
3. 每个阶段输出都做 JSON 化。
4. 每一步都保留人工审核点。
5. 最后一步才生成视频。

这样后续做第 4 集、第 5 集时，流程就不需要再靠人工临时口头纠偏。
