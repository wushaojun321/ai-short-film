# AI短剧生产流程固化摘要

## 1. 目标定义

这套流程不是“把剧本文本直接丢给视频模型生成成片”，而是：

1. 输入总剧本。
2. 拆成分集剧本。
3. 每集先拆成片段。
4. 每个片段再拆成多个短镜头。
5. 先建立全剧资产规格书，列出角色、造型、场景、模板组、关键帧需求，但不立即全部生成。
6. 每个镜头绑定资产、台词、机位、声音和连续性约束。
7. 在镜头生成前做一次缺失资产检查，只生成当前镜头确实缺少的资产。
8. 新生成资产先审核，审核通过后再调用视频模型生成单镜头。
9. 人工审核镜头。
10. 通过后按集拼接。

核心目标不是“能生成”，而是：

- 角色一致。
- 场景一致。
- 声音一致。
- 镜头语言统一。
- 片段之间衔接自然。
- 后续可批量化生产。

## 2. 已验证的主流程

### 2.1 总流程

1. 研读总剧本，提炼人物关系、情绪线、权力关系、阶段冲突。
2. 拆成分集剧本，尽量忠于原稿，不随意改写台词和剧情。
3. 每集先拆 `beat / 片段`，不要直接拆成长镜头。
4. 每个片段再拆成 `3-8 秒` 的短镜头组。
5. 先输出资产需求设计，不立即全量生成资产。
6. 在实际镜头执行前，检查当前镜头缺失哪些资产。
7. 只生成当前镜头所缺资产，并先做资产审核。
8. 资产确认后，把真实 `source_url` 填入镜头提交体。
9. 逐镜生成。
10. 每生成一批镜头先做片段预览。
11. 通过后再合成整集。

### 2.1.1 待真实项目测试后固化的蓝图层

当前项目正在把解析阶段收敛到 `ProductionBlueprint` 中间真相层。该层的目标是把“剧情理解”和“制作执行”分开：

- 剧情理解：剧本解析、全剧规划、分集蓝图、每集资产需求。
- 资产规划：人物圣经、人物阶段资产、场景/道具圣经。
- 执行派生：从蓝图派生 `Episode`、`Asset`、后续 `Shot` 和视频请求。

下一轮真实项目测试通过后，再继续把本地实跑验证过的结构固化进蓝图：

- `script_analysis`：主要人物、人物关系、主线/副线、关键节点、分集切点、角色状态变化线、原稿风险点。
- `beats`：片段编号、功能、场景、人物、开头状态、结尾落点、前后衔接方式、片段级资产需求。
- `character_cores / character_looks`：人物核心身份、人脸锁定、音色锁定、阶段造型。
- `scene_packages / scene_views`：场景包、空间坐标、不同视角和时间/状态变体。
- `shot_asset_bindings`：镜头级角色、造型、场景视角、说话人、静默角色、关键帧和尾帧策略。
- `seedance_request_packages`：真实提交给 Seedance 的文本、参考图、ratio、duration、return_last_frame 等完整请求体。

注意：这些是后续优化方向，不要在没有真实项目回归样例前一次性全量实现。

### 2.2 每集生产顺序

固定顺序应为：

1. 分集剧本。
2. 连续性约束。
3. 片段拆分。
4. 镜头脚本。
5. 资产规格与需求清单。
6. 正式提交提示词审核。
7. 镜头执行前缺失资产检查。
8. 只生成缺失资产。
9. 资产审核。
10. 视频生成。
11. 审核与重生。
12. 合成成片。

不要倒过来做。不要先生成一批图，再反过来猜镜头。

### 2.3 待产品化的提交前校验

本地实跑确认，很多生成问题不应只靠改提示词解决，而应在视频提交前做确定性校验。后续应增加 `ShotPreflightValidator`，至少检查：

- 每个镜头必须绑定出镜角色资产。
- 每个镜头必须绑定场景资产。
- 有台词时必须声明唯一说话人和 mouth-sync 角色。
- 多人镜必须区分 `speaker` / `listener` / `silent_character`。
- 使用 `last_frame` 时仍必须绑定角色和场景资产。
- `last_frame` 只能辅助连续性，不能作为身份锚点。
- `look_id` 必须可回溯到 `character_id`。
- 台词字数必须能被镜头时长承载。
- 片段首镜、三人以上关系镜、复杂动作镜、身份易混镜需要标记关键帧需求。

## 3. 分镜设计规则

### 3.1 不再使用“一个剧情点=一个长镜头”

之前的问题：

- 镜头过长，常达到 `10-15 秒`。
- 模型要在同一镜头里同时完成站位、对白、动作、情绪、运镜。
- 最终容易变成“近景人像镜头 + 口型混乱 + 身份漂移”。

调整后规则：

- 一个剧情片段拆成多个短镜头。
- 单镜头尽量控制在 `3-8 秒`。
- 少量建立镜或过渡镜可到 `10-12 秒`。
- 只有确实必要时才使用接近模型上限的镜头时长。

### 3.2 每个片段的镜头组成建议

一个完整片段通常由这些镜头构成：

- 建立镜：交代空间和人物位置。
- 动作镜：交代事件发生。
- 关系镜：交代谁压谁、谁看谁。
- 反应镜：交代情绪落点。
- 过渡镜：连接场景或节奏。
- 悬念镜：落到一句台词或一个动作。

### 3.3 景别配比规则

为避免全是近景，强制拉开景别层次：

- 建立镜：大全景 / 全景 / 中景。
- 关系镜：双人中景 / 三人中景 / 过肩。
- 对白镜：中景 / 中近景。
- 反应镜：中近景 / 近景。
- 重要信息点：短特写。

不能整集都堆成：

- 近景正拍。
- 脸部特写。
- 两人站着说话。

### 3.4 机位必须写清楚

每个镜头必须明确：

- 景别。
- 机位方向。
- 机位高度。
- 运镜方式。
- 时间分段动作。

例如：

- 沿中轴线正面推进。
- 门口中景固定。
- 过肩关系镜。
- 略低机位但不夸张仰拍。
- `0-2秒 / 2-5秒 / 5-8秒` 分阶段描述动作和视线变化。

不能只写“生成一个宫廷镜头”。

## 4. 连续性控制规则

### 4.1 连续性不是附属项，是主约束

连续性必须在每集开工前先写清楚，至少包括：

- 人物服装是否变化。
- 发型是否变化。
- 伤势是否延续。
- 蒙眼、佩饰、道具是否还在。
- 人物固定站位。
- 时间光线。
- 情绪承接。
- 谁从哪边入镜，谁看向哪边。

### 4.2 固定空间坐标

之前镜头生硬的重要原因之一，是人物每镜都在乱跳位。

现在的规则是先写死空间坐标，例如：

- 大厅主位。
- 大厅门口。
- 庭院中央练剑区。
- 廊下。
- 门口过渡位。
- 东侧 / 西侧 / 北侧 / 南侧人物活动区。

镜头切换时，不允许无理由改站位。

### 4.3 关键连续性必须写成硬规则

例如第二集验证出的规则：

- `S01-S03`：李云湘必须戴黑色蒙眼纱布。
- `S04`：当镜摘下黑色纱布。
- `S05` 之后：禁止再出现纱布。

这种规则必须写进连续性文档和镜头提示词，不能只靠记忆。

## 5. 资产策略

### 5.0 先约定资产规格，不立即生成

资产流程现在明确成两步：

1. 先定义全剧和单集所需的资产规格。
2. 不立即生成全部资产，只在镜头执行前检查缺失资产，再按需生成。

也就是说：

- 分解剧本后，先列出需要哪些角色资产、造型资产、场景资产、模板资产、关键帧资产。
- 这一步的目标是把需求列全、规则定清，不是马上出图。
- 到真正执行某一镜头前，再检查这个镜头需要的资产是否齐全。
- 只有缺失的资产才进入生成与审核。

这样做的好处：

- 保证全局资产规划一致。
- 避免一上来生成大量后续可能用不到的图片。
- 可以边拍边补，但不会失去全局一致性。
- 更适合长剧、多集批量生产。

### 5.0.1 建议先列出的资产类型

分解剧本后，至少先输出这些需求表：

- 角色核心资产需求表
- 角色造型资产需求表
- 场景资产需求表
- 模板群演资产需求表
- 关键片段关键帧需求表

### 5.0.2 每次执行前的资产检查

在某一镜头真正提交视频生成前，必须执行：

1. 检查该镜头需要绑定的角色资产是否存在。
2. 检查该镜头需要绑定的场景资产是否存在。
3. 检查该镜头是否需要关键帧资产。
4. 检查缺失资产是否已审核通过。

若缺失：

- 先生成缺失资产。
- 先做资产审核。
- 审核通过后再执行该镜头的视频生成。

### 5.1 先以 Seedream 生成人物母图

已经验证：

- 纯 `Seedream` 直出的人物图，更适合作为 `Seedance` 参考图。
- `Gemini -> Seedream` 虽然能帮助找脸型方向，但有被 `Seedance` 当作敏感或不稳定输入的风险。

当前更稳策略：

- 最终用于视频生成的人物母图，优先使用纯 `Seedream` 产物。
- Gemini 只作为提示词参考和脸型方向参考，不直接作为视频参考资产链的核心。

### 5.2 人物资产不能只写“帅哥/美人”

角色相似度高的根因，是提示词写成了泛化古装脸。

主角色资产必须具备：

- 明确脸型。
- 明确肤色。
- 明确骨相。
- 明确发型。
- 明确配饰系统。
- 明确身份气质。
- 明确排斥项。

例如：

- 谢风凌：黑衣、武人、深肤、硬骨相、旧伤、绝不能像皇帝或旧宠。
- 李睿：冷白、规整、帝王冠、绝不能像侍卫。
- 顾文池：白衣执扇、男性旧宠、柔骨相、绝不能变女性或侍女。
- 李云湘：深绛红长公主、女性掌权者、危险感、绝不能生成普通古偶温柔女主。

此外，人物资产不能只停留在面部设定，还应包括：

- 正面脸
- 侧面脸
- 半身标准像
- 全身标准像
- 不同剧情状态下的服装
- 发型
- 妆造
- 常用配饰
- 常用道具

也就是人物资产要覆盖：

- 面部识别
- 身体比例
- 服装系统
- 发型妆造系统
- 道具系统

### 5.2.1 人物资产按场景和阶段拆分

人物资产不能用“一个角色一张图贯穿整部剧”的方式处理。

解析剧本时，应按以下维度列出人物资产：

- 角色本名。
- 适用场景。
- 剧情阶段或造型阶段。
- 服装状态。
- 伤势 / 妆发 / 道具状态。
- 视角要求。

同一角色只要出现明显造型变化，就应拆成独立人物资产，例如：

- `李云湘-朝堂掌权期-深绛宫装`
- `李云湘-寝殿私下期-素色便服`
- `谢风凌-战损阶段-黑衣负伤`
- `顾文池-宴会试探期-白衣执扇`

分镜绑定人物资产时，必须选择与当前镜头场景和剧情阶段匹配的人物造型资产，不能默认使用同一角色的通用资产。

### 5.2.2 人物形象资产必须包含三张独立视角图

每个正式人物造型资产生成时，必须包含：

- 面部特写图。
- 全身正面图。
- 侧面视角图。

三类视角必须是三张不同图片，不允许拼成三宫格、分屏图或同图多视角。要求是同一位真人演员、同一套造型、同一阶段状态，作为后续视频生成的稳定身份参考。不能只生成单张全身图，也不能生成插画式角色设定图。

同一角色的多个阶段造型在前端按 `asset_package` 集中成一个人物资产包，点击后展开该角色不同阶段的资产卡片子集。

### 5.3 模板资产要做“组”，不是单模板

之前侍女、侍卫等群演资产太像。

现在规则：

- 侍女至少 `A/B/C`
- 侍卫至少 `A/B/C`
- 近臣至少 `A/B`
- 刺客至少 `A/B`
- 面首至少 `A/B/C`

也就是模板资产要做成“多变体模板组”，不能全剧共用同一张模板脸。

### 5.4 场景资产不是一张图，而是场景包

每个主场景至少应提前约定这些资产需求：

- 全景建立图
- 主活动区视角
- 门口 / 廊下 / 入口视角
- 对峙视角
- 过渡视角
- 昼夜或时间状态变体

这样后续镜头切换时，不会因为缺视角而临时乱生。

### 5.5 关键片段应预留关键帧资产需求

关键帧不要求全镜头生成，但应提前在资产需求里标出：

- 每个片段首镜
- 三人以上关系镜
- 身份容易混淆的镜头
- 动作复杂镜头
- 重要悬念镜

这些镜头需要关键帧资产时，先在需求表里约定，执行前再生成。

## 6. 视频生成时的参考图策略

### 6.1 每一个镜头都必须绑定角色资产和场景资产

已经验证出的结论：

- 每一个镜头都必须绑定角色资产。
- 每一个镜头都必须绑定场景资产。
- `last_frame -> next first_frame` 只能作为连续性辅助，不能代替角色和场景锚定。
- 如果不重复绑定角色和场景，连续镜头会出现身份漂移、性别漂移和空间细节漂移。

典型案例：

- 第三集后半段 `S10-S14`，只靠尾帧会把顾文池漂成女性形象。

### 6.2 推荐的参考策略

片段首镜：

- 使用角色资产 + 场景资产作为 `reference_image` 锚定。
- 可不使用 `first_frame`。

片段内普通承接镜：

- 仍然必须绑定当前镜头的角色资产 + 场景资产。
- 在此基础上，再使用上一镜 `last_frame` 作为辅助连续性参考。
- 不能只传 `first_frame + text`。
- 批量生成时必须按镜头顺序执行，不能把同一集所有镜头并发提交；否则后一镜拿不到上一镜 `last_frame`。
- `last_frame` 必须转存为长期可用素材地址，不能依赖模型平台返回的临时链接。

角色身份容易漂的关系镜：

- 每镜重新绑定多角色资产。
- 每镜重新绑定场景资产。
- 尾帧如果使用，只能作为补充，不能取代角色和场景绑定。

### 6.3 什么场景不能只靠尾帧

以下场景不适合纯尾帧链：

- 三人以上关系镜。
- 人物正式亮相镜。
- 身份容易混淆的双男 / 三人镜。
- 片段转场首镜。
- 角色说话权必须明确的对白镜。
- 性别、服装、身份容易串位的镜头。

按当前规则，应进一步明确为：

- 没有任何镜头允许只靠尾帧。
- 有些镜头可以在“角色资产 + 场景资产”之外，再额外叠加尾帧辅助。
- 同片段后续镜头默认可以叠加上一镜尾帧；片段首镜、明显换场、时间跳跃默认不叠加，除非连续性字段明确要求。

### 6.4 合并阶段的轻量连续性处理

分集合并不是简单拼接文件。合并前应统一每个镜头的视频规格，并做轻量处理：

- 每个镜头视频统一竖屏尺寸、编码、音频采样率，避免 concat 后卡顿或音频跳变。
- 每个镜头音频做极短淡入淡出，降低硬切噪声。
- 片段切换且不是同场连续动作时，可插入极短黑场缓冲。
- 同场对白、动作连续、视线承接镜仍保持硬切，避免短剧节奏被拖慢。

## 7. 提示词写法规则

### 7.1 统一结构

每个镜头提示词应包含：

1. 全局视觉风格。
2. 场景参考。
3. 人物参考。
4. 镜头功能。
5. 固定站位。
6. 景别与机位。
7. 运镜。
8. 时间分段动作。
9. 台词与说话人。
10. 声音风格。
11. 反向约束。

### 7.2 台词必须明确“谁说”

已验证的问题：

- 只写对白内容，不写说话权，模型会把台词分给错误人物。

现在规则：

- 明确写“这句台词只能由谁发声”。
- 明确写“谁不能开口”。
- 必要时把镜头改成单人说话镜。

例如：

- 谢风凌唯一发声。
- 顾文池只做无声反应。
- 顾文池不得张嘴。

### 7.3 台词必须绑定表演信息

每句台词不只保存文本，还必须保存表演指令：

- `speaker`：唯一说话人。
- `text`：台词原文，不能改写。
- `emotion`：本句情绪，例如压抑怒意、强忍哽咽、冷静试探。
- `delivery`：声音表演，例如低声、短促、语速偏慢、尾音压住。
- `action`：说话时同步发生的身体动作。
- `expression`：说话时的面部表情和眼神。

视频提示词必须把这些信息放入“台词与表演”块，避免只生成有口型但没有表演逻辑的对白镜。

### 7.4 角色音色资产化

人物资产除了视觉 prompt，还应保存 `voice_profile`：

- 年龄感。
- 性别。
- 音色质感。
- 语速。
- 情绪基线。
- 禁止变化项。

每次生成视频时，只要镜头中出现该角色，就把对应 `voice_profile` 带入提示词。这样同一角色跨镜头的声音基准保持一致。

示例：

```text
李云湘固定音色：成年女性，低沉清冷，吐字清晰，语速偏慢，情绪克制，带掌权者压迫感。不要少女撒娇音，不要尖叫失真。
```
- 嘴型只能出现在谢风凌脸上。

### 7.3 角色身份必须在提示词里重复绑定

尤其是多角色镜头，必须直接写：

- 李云湘是女性长公主。
- 谢风凌是黑衣男性侍卫。
- 顾文池是白衣执扇成年男性旧宠。

并加排斥项：

- 顾文池不得生成女性。
- 不得生成侍女。
- 不得生成公主。
- 人物身份、服装、性别不能串。

### 7.4 不要让模型自己理解“观察关系”

例如：

- “顾文池在远处观察谢风凌”

不能只写这一句，要写成：

- 顾文池位于前景右侧。
- 身体和头部朝左前方练剑区。
- 目光明确落在远处练剑的谢风凌身上。
- 不得看镜头。
- 不得看向右侧画外。

## 8. 声音策略

### 8.1 当前策略

为了避免口型错位，当前实践上优先使用：

- `generate_audio=true`

原因：

- 后期纯补配音，口型容易对不上。
- 镜头本身有对白时，模型直接生音频更容易让嘴型和台词同步。

### 8.2 但音色一致不能只靠模型自由发挥

后续项目里要固化：

- 每个主角色有固定音色描述词。
- 每句台词都带说话人。
- 重要角色建议后续补 `reference_audio` 支持。

当前已使用的音色锚点方式是：

- 李云湘：冷静、低缓、克制、威压感强。
- 谢风凌：低沉、克制、寡言。
- 顾文池：温柔、带笑意、轻讽、会讨好人。
- 李睿：平静、克制、贵气、藏锋。

## 9. 人工审核机制

### 9.1 审核必须分层

不能最后只看整集。

应该分三层：

1. 资产审核。
2. 镜头审核。
3. 片段审核。
4. 整集审核。

### 9.2 镜头审核重点

每次审镜头优先看：

- 角色是不是对的人。
- 台词是不是对的人在说。
- 站位是不是符合分镜。
- 视线是不是对。
- 性别有没有漂。
- 服装有没有串。
- 空间方向有没有乱。
- 是否又退化成单一近景。

### 9.3 不要整集重做，按问题片段重生

当前已验证更高效的方式是：

- 找出具体有问题的镜头。
- 如果是片段身份漂移，就按片段整体重做。
- 不要全集重生。

例如第三集：

- `S10-S14` 片段整体重做比单改一镜更合理。

## 10. 当前已验证的常见失败模式

### 10.1 角色相似度过高

原因：

- 主角色提示词写成古装通用帅哥/美人。

解决：

- 强骨相差异。
- 强肤色差异。
- 强服装系统差异。
- 强发型配饰差异。
- 强排斥项。

### 10.2 多角色镜头里身份串位

原因：

- 只用尾帧接续。
- 没有逐镜重新绑定角色。

解决：

- 在关键关系镜里每镜重绑角色资产。

### 10.3 说话人错误

原因：

- 双人同框镜里只写对白内容，没有写说话权。

解决：

- 写死唯一说话人。
- 必要时改成单人说话镜。

### 10.4 场景切换生硬

原因：

- 直接从一个片段硬切到另一个片段。

解决：

- 增加过渡镜。
- 加门口镜、方向镜、建立镜。

### 10.5 整集都是近景

原因：

- 没做景别配比。

解决：

- 强制每片段有建立镜、关系镜、落点镜。

## 11. 当前推荐的项目固化规则

后续代码层建议直接固化为这些模块：

1. `script_parser`
   - 总剧本提取。
   - 分集拆分。

2. `beat_planner`
   - 每集拆 `beat / 片段`。

3. `shot_planner`
   - 每片段拆成短镜头。
   - 自动补建立镜、过渡镜、落点镜。

4. `continuity_manager`
   - 维护服装、伤势、蒙眼、道具、站位、时间、朝向等状态。

5. `asset_bible`
   - 人物资产规格。
   - 造型资产规格。
   - 场景资产规格。
   - 模板组资产规格。
   - 关键帧资产规格。
   - 资产审核状态。

6. `prompt_builder`
   - 按固定结构生成导演式镜头提示词。

7. `reference_strategy_resolver`
   - 判断当前镜头如何组合：
     - 角色资产
     - 场景资产
     - 尾帧辅助
   - 但无论如何组合，每镜都必须包含角色资产和场景资产绑定

8. `shot_executor`
   - 调用视频模型。
   - 执行前先做缺失资产检查。
   - 保存 `request / response / status / output / last_frame`

9. `review_loop`
   - 人工审核结果记录。
   - 失败镜头重生。

10. `episode_assembler`
    - 合成片段预览。
    - 合成整集。
    - 生成 manifest。

## 12. 当前最重要的产品决策

后面项目固化时，应坚持这几个原则：

1. 忠于原稿，不做过度改编。
2. 先分片段，再分短镜头，不再直接做长镜头。
3. 角色资产先定稿，再做视频。
4. 每一个镜头都必须绑定角色资产和场景资产，关键关系镜更不能只靠尾帧。
5. 台词必须绑定说话人。
6. 片段之间必须有过渡镜。
7. 每集先出预览，再出整集。
8. 发现问题优先重做问题片段，不整集推倒重来。

## 13. 当前可直接沿用的工作结论

已经验证通过、后续可以直接沿用：

- 使用 `Seedream` 直出人物母图，再给 `Seedance` 做参考。
- 每集先写连续性约束文档。
- 每集先输出资产规格与需求表，不立即全量生成资产。
- 每集先出正式提交审核版，再填真实 URL。
- 每镜执行前先检查缺失资产，再按需生成和审核。
- `Seedance` 中：
  - 每镜都绑定角色资产和场景资产
  - 片段首镜优先 `reference_image`
  - 片段内承接镜可额外叠加 `last_frame -> first_frame` 做辅助连续性
- 关键关系镜需要反复重绑多角色资产。
- 模板群演资产必须做多变体组。
- 有对白镜头优先 `generate_audio=true`
- 整集前先合成片段预览。

## 14. 下一步项目落地方向

如果要把这套流程正式做进项目，下一步应优先做：

1. 把“分集 -> 片段 -> 短镜头”的数据结构固定下来。
2. 把连续性状态做成结构化对象，而不是散落在文案里。
3. 把角色/场景/造型/模板资产做成统一资产索引表。
4. 把镜头提交策略做成规则引擎。
5. 把审核和重生流程做成可追踪的状态机。

这样后面做第 4 集、第 5 集时，就不需要再靠人工临时约束，而是直接按系统规则生产。

## 15. 已落地：低 token 剧本解析链路

当前初始化解析不采用串行多 Agent，而是采用“原文索引 + 单次综合规划 + 后端归并派生”的折中生产方案：

1. `ScriptIndexer` 先把原始剧本切成 `script_blocks`，保留行号、块类型、说话人和原文位置。
2. `ScriptProductionPlanAgent` 一次输出 JSONL 蓝图行，包括 series、episode、character、character_variant、scene、scene_variant、prop、prop_variant、ignore 和 warning。
3. `EpisodeMaterialBuilder` 由后端根据 start_block / end_block 回填 `Episode.script_excerpt`，保证对白不会在摘要阶段丢失。
4. 后端把 JSONL 资产注册表归并为 `ProductionBlueprint`，再派生当前前端兼容的 `Episode` / `Asset` 记录。
5. 资产图片提示词不再二次调用 LLM，而由确定性 `asset_prompt_builder` 根据资产字段生成最终提交给 Seedream 的提示词。

这套方案的目标是降低重复上下文和输出过长风险。`SeriesPlannerAgent`、`EpisodeSplitterAgent`、`AssetExtractorAgent` 等独立 Agent 仍可作为后续演进方向，但不再写作当前已落地实现。

人物资产一致性规则：

- 同一人物的不同状态、不同场景造型视为同一人物资产包，而不是互不相关的独立角色。
- 同一 `asset_package` 内所有造型共享同一 `face_identity`，不同场景造型使用同一面部形象参考生成，只改变服装、妆发、伤势、道具和场景状态。
- 除非剧本明确出现面部受伤、毁容、年龄跨度或易容伪装，否则不得改变同一人物的脸型、骨相、五官比例、肤色和标志性面部特征。
- 每个角色资产包增加 `distinctive_traits` 和 `avoid_similar_to`：前者固定 3-5 个可视面部差异点，后者列出容易混淆的角色或排斥特征，防止不同角色生成成近似脸。
- 每个人物阶段资产增加 `look_lock`：锁定该阶段的发型/发际线、胡须状态、服装颜色款式材质、领口袖口腰带、配饰、伤势和随身道具。面部、全身、侧面三张独立图只能改变景别和角度，不能重设造型。

这个改动保留现有 `/generate/projects/{project_id}/parse-script` 入口和前端流程，只改变后端任务内部的数据来源与落库方式。
